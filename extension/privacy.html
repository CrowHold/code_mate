Code_Mate

Privacy Policy · Last updated 2026-05-13

Short version: Code_Mate collects no personal data. Nothing you look up is sent anywhere except, optionally and only on your request, to your own Anthropic API key.

What Code_Mate does not collect

  • No analytics or usage tracking of any kind
  • No telemetry on which terms you look up
  • No crash reports sent to a remote server
  • No account, login, or profile
  • No browsing history
  • No data sold, shared, or monetised

What stays on your device

All extension state is stored in chrome.storage.local, which is private to your browser and your machine. This includes:

  • Indexed glossary — a pre-processed version of the bundled data/glossary.json file, stored locally for fast lookup.
  • AI cache — definitions generated via the AI fallback are cached locally under the key glossary_ai_cache. They never leave your device after generation.
  • Your Anthropic API key — if you choose to add one in Settings. Stored in chrome.storage.local; never transmitted except to api.anthropic.com as the Bearer token in your own API requests (see below).
  • The Sonnet-for-ELI25 preference — a boolean toggle stored alongside your key.

AI fallback and the Anthropic API

The AI fallback feature is opt-in and requires you to supply your own Anthropic API key. When you click "Generate with AI", Code_Mate's service worker sends the selected term to https://api.anthropic.com/v1/messages using your key as the Bearer token.

This means the selected term is sent to Anthropic's servers on your behalf, under your account's API usage. Anthropic's own privacy policy applies to those requests: anthropic.com/legal/privacy.

If you do not add an API key, no network requests are made by Code_Mate beyond loading the extension's own bundled files and the web fonts used in the bubble UI.

Web fonts

The bubble UI uses Instrument Sans and JetBrains Mono, loaded via Google Fonts CDN (fonts.gstatic.com). This is a standard browser resource fetch — your IP address is visible to Google's CDN servers, the same as when any website loads a Google Font. No Code_Mate-specific data is attached to these requests.

If you prefer to avoid this fetch, both fonts have system-font fallbacks (the UI remains fully functional with -apple-system / Menlo / Consolas). A future version may bundle font files locally to eliminate the CDN fetch entirely.

Permissions explained

  • contextMenus — adds the "Define with Code_Mate" item to your right-click menu. Required for the core trigger.
  • storage — stores the glossary index and AI cache locally. All data stays on your device.
  • scripting — injects the bubble UI into the active tab when you trigger a lookup. Nothing is injected proactively.
  • <all_urls> host permission — required so the right-click trigger works on any page you're reading, not just a whitelist. No content from any page is read or transmitted.
  • https://api.anthropic.com/* host permission — required to call the Anthropic API from the MV3 service worker. Only used when you explicitly click "Generate with AI" with a key saved.

Open source

Code_Mate is open source. You can inspect every line of code at github.com/CrowHold/code_mate. The extension performs no obfuscation and no minification — what you see in the source is exactly what runs.
